# v1.5.0 — Safer lifecycle, DNS visibility, and adaptive UI

> **Language / Idioma:** English first, Português (Brasil) below.

Release date: 2026-07-10

## Highlights

- Shows a persistent, localized warning when the CLI cannot configure system DNS, preventing a silent DNS-leak risk.
- Adds color-coded latency badges and normalizes malformed location capitalization from CLI output.
- Prevents false disconnect states, duplicate log windows, and popout overflow on short displays.
- Exposes the opt-in systemd-managed tunnel mode in Settings → Advanced; the default direct CLI lifecycle remains unchanged.
- Strengthens CI with immutable action SHAs, a locked Markdown toolchain, shell validation, QML validation, and 29 parser tests.

## Compatibility

- Requires DankMaterialShell `>= 1.4.0`.
- Requires `adguardvpn-cli` in `PATH` or a configured binary path.
- The systemd-managed mode remains optional and requires the separately installed `/usr/local/sbin/adguardvpn-dms-control` helper.
- No existing setting changes behavior by default.

## Validation

- All 29 parser tests pass.
- i18n key validation passes for 22 locales; DNS safety warnings are translated in every locale.
- Markdown, ShellCheck, Actionlint, and QML syntax checks pass.
- The plugin reloads successfully and reports `loaded` in DMS.

## Português (Brasil)

Data de lançamento: 2026-07-10

### Destaques

- Mostra um aviso persistente e localizado quando o CLI não consegue configurar o DNS do sistema, evitando um risco silencioso de vazamento DNS.
- Adiciona badges de latência por cor e normaliza capitalização incorreta de localizações recebidas do CLI.
- Evita estados falsos de desconexão, janelas duplicadas de log e estouro do popout em telas baixas.
- Expõe o modo opt-in de túnel gerenciado pelo systemd em Configurações → Avançado; o ciclo de vida direto pelo CLI continua sendo o padrão.
- Reforça o CI com SHAs imutáveis, toolchain Markdown travada, validação de shell/QML e 29 testes de parser.

### Compatibilidade

- Requer DankMaterialShell `>= 1.4.0`.
- Requer `adguardvpn-cli` no `PATH` ou caminho configurado.
- O modo gerenciado pelo systemd continua opcional e exige o helper `/usr/local/sbin/adguardvpn-dms-control`, instalado separadamente.
- Nenhuma configuração existente muda de comportamento por padrão.

### Validação

- Todos os 29 testes de parser passam.
- A validação i18n passa para 22 locales; avisos de segurança DNS estão traduzidos em todos eles.
- Checks de Markdown, ShellCheck, Actionlint e sintaxe QML passam.
- O plugin recarrega com sucesso e aparece como `loaded` no DMS.

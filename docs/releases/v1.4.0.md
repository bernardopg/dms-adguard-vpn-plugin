# v1.4.0 — Real timeouts, visible inputs, retry & tests

> **Language / Idioma:** English first, Português (Brasil) below.

Release date: 2026-06-12

## Highlights

This is the largest quality release since 1.3.0: four critical fixes, a resilience
upgrade, full keyboard accessibility, and the first automated test suite.

### Critical fixes

- **Command timeouts now actually work.** The value previously passed to
  `Proc.runCommand` was the debounce delay, not the timeout, so every CLI call was
  silently capped at the 10 s default. Connects now get 60 s, `disconnect`/`license`
  30 s, reads 15 s, and a timeout (exit 124) shows a dedicated error message.
- **Locations search and direct-destination fields were invisible in production.**
  `Layout.fillWidth` attached properties inside plain `Column` positioners collapsed
  them to zero width. Both are now full-width `DankTextField`s with icons — verified
  with before/after screenshots.
- **The destination field no longer starts empty** when a default location is saved:
  it seeds from a reactive `pluginData` binding on every popout open instead of a
  one-shot copy that raced plugin-data loading.
- **Saving any setting no longer triggers a full CLI refresh cascade.** Favoriting a
  location used to restart timers and spawn 4+ CLI processes; timers now restart only
  when polling settings change and CLI availability is rechecked only when the binary
  path changes.

### Resilience

- Auto-reconnect retries up to 3 times with 5 s / 15 s / 45 s backoff and reports when
  it gives up.
- Status polling backs off to ≥30 s while the CLI is unavailable.
- Connect/location parser no longer mistakes lowercase words ("to", "in") for ISO codes.
- Opening the tunnel log no longer reports a false failure: blocking terminal emulators
  are launched in the background.

### UI & accessibility

- Popout scrolling via the shell's `DankFlickable` — smooth momentum, auto-hiding
  overlay scrollbar with a reserved channel so it never covers content.
- "Show more" paging replaces the hidden 8-item cap in the locations list; the
  "Showing X/Y" counter is now honest.
- Full keyboard navigation: action buttons, location cards, and favorite stars are
  Tab-focusable, respond to Enter/Space, and expose `Accessible` roles.
- "Not logged in" banner with the exact login command and a copy button.
- Hero chips for the active tunnel interface (TUN) and the SOCKS endpoint.
- Enter connects from the destination field and applies DNS from the DNS field.

### Testing & docs

- `scripts/test-parsers.mjs`: 26 dependency-free unit tests for the status, license,
  config, and locations parsers, wired into CI.
- README refreshed for registry listing: new cover banner, badges, updated feature
  matrix, documented `bypassMultiRouteCheck`, and a new screenshot composed from real
  captures of the current popout.

## Compatibility

- DankMaterialShell `>= 1.4.0` (unchanged).
- No breaking settings changes. Legacy `favoriteLocationIsos` favorites are migrated
  and persisted to `favoriteLocationTargets` on first load.

## Português (Brasil)

Data de lançamento: 2026-06-12

### Destaques

Maior release de qualidade desde a 1.3.0: quatro correções críticas, upgrade de
resiliência, acessibilidade completa de teclado e a primeira suíte de testes.

#### Correções críticas

- **Timeouts de comando agora funcionam de verdade.** O valor passado ao
  `Proc.runCommand` era o debounce, não o timeout — toda chamada ficava limitada aos
  10 s padrão. Connects agora têm 60 s, `disconnect`/`license` 30 s, leituras 15 s, e
  timeout (exit 124) mostra mensagem dedicada.
- **Campos de busca e destino direto estavam invisíveis em produção** por
  `Layout.fillWidth` dentro de `Column` puro; agora são `DankTextField`s em largura
  total com ícones (verificado por screenshots).
- **O campo de destino não inicia mais vazio** com localização padrão salva: seed por
  binding reativo a `pluginData` a cada abertura do popout.
- **Salvar settings não dispara mais cascata completa de refresh do CLI.**

#### Resiliência

- Auto-reconnect com até 3 tentativas (backoff 5 s / 15 s / 45 s) e aviso de desistência.
- Polling recua para ≥30 s com CLI indisponível.
- Parser não confunde palavras minúsculas com códigos ISO.
- Abrir o log do túnel não reporta mais falha falsa.

#### UI e acessibilidade

- Scroll do popout via `DankFlickable` com scrollbar overlay em canal reservado.
- Paginação "Mostrar mais" no lugar do teto oculto de 8 itens; contador honesto.
- Navegação completa por teclado com foco visível e roles `Accessible`.
- Banner "Sessão não iniciada" com comando copiável; chips de interface/SOCKS no hero;
  Enter conecta/aplica DNS.

#### Testes e documentação

- `scripts/test-parsers.mjs`: 26 testes sem dependências, integrados ao CI.
- README renovado: capa nova, badges, matriz de recursos, `bypassMultiRouteCheck`
  documentada e screenshot real do popout atual.

### Compatibilidade

- DankMaterialShell `>= 1.4.0` (sem mudança).
- Sem quebra de settings. Favoritos legados migrados e persistidos no primeiro load.

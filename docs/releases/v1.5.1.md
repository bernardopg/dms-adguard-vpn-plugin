# v1.5.1 — Location connections and managed status

> **Language / Idioma:** English first, Português (Brasil) below.

Release date: 2026-07-10

## Highlights

- Restores quick connect for every location row by sending the city format accepted by `adguardvpn-cli` instead of the internal `city, country` display key.
- Keeps systemd-managed tunnels visibly connected when the unprivileged CLI returns exit code 10 with `VPN active, but status unavailable`.
- Avoids empty connected summaries when location metadata is unavailable.

## Compatibility

- Requires DankMaterialShell `>= 1.4.0`.
- Requires `adguardvpn-cli` in `PATH` or a configured binary path.
- The systemd-managed mode remains optional and requires the separately installed `/usr/local/sbin/adguardvpn-dms-control` helper.
- No settings migration is required.

## Validation

- All 31 parser tests pass.
- i18n key validation passes for 22 locales.
- Markdown and QML syntax validation pass.
- Live validation connected specifically to São Paulo in TUN mode, confirmed `tun0` active, and verified the DMS UI displayed the connected state.

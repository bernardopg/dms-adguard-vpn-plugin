# AdGuard VPN Plugin v1.3.4

This release hardens startup behavior, fixes location targeting, and makes the repository documentation fully bilingual with English as the primary language and Portuguese (Brazil) as the companion language.

## Highlights

- Auto-connect on startup now waits for both initial status and configuration reads before connecting, preventing SOCKS setups from being blocked by TUN-only preflight checks.
- Enabling Auto Connect after startup no longer triggers an immediate connection outside the startup window.
- Runtime preflight now disconnects through the configured `adguardBinary` instead of a hardcoded `adguardvpn-cli` command.
- Location quick-connect and favorites now target the specific city/country row instead of collapsing every location to its country ISO code.
- Boolean settings loaded from string storage now handle `"false"`, `"0"`, `"off"`, and `"no"` correctly.
- Documentation, release notes, issue templates, PR template, workflow labels, and the release agent are now bilingual EN/PT-BR.
- GitHub Actions now runs the i18n key parity check in addition to Markdown and QML validation.

## Compatibility

- Requires DankMaterialShell `>= 1.4.0`.
- Requires `adguardvpn-cli` available in PATH or configured through `adguardBinary`.

## Validation

- `node scripts/check-i18n-keys.mjs`
- `bash scripts/lint-markdown.sh`
- `bash scripts/validate-qml.sh`
- GitHub issue/workflow YAML syntax validation

## Notes

- Previous release: `v1.3.3`

## Português (Brasil)

Esta versão fortalece o comportamento de startup, corrige o alvo de localizações e transforma a documentação do repositório em bilíngue, com inglês como idioma principal e português (Brasil) como idioma complementar.

### Destaques

- Auto-connect no startup agora espera as leituras iniciais de status e configuração antes de conectar, evitando que setups SOCKS sejam bloqueados pelo preflight exclusivo de TUN.
- Ativar Auto Connect depois do startup não dispara conexão imediata fora da janela de inicialização.
- O preflight de runtime agora desconecta usando o `adguardBinary` configurado em vez de chamar `adguardvpn-cli` hardcoded.
- Quick-connect e favoritos por localização agora miram a linha específica cidade/país em vez de reduzir tudo ao ISO do país.
- Settings booleanas carregadas como string agora tratam `"false"`, `"0"`, `"off"` e `"no"` corretamente.
- Documentação, release notes, issue templates, PR template, labels do workflow e agente de release agora estão em EN/PT-BR.
- GitHub Actions agora roda o check de paridade i18n além das validações de Markdown e QML.

### Compatibilidade

- Requer DankMaterialShell `>= 1.4.0`.
- Requer `adguardvpn-cli` no PATH ou configurado por `adguardBinary`.

### Validação

- `node scripts/check-i18n-keys.mjs`
- `bash scripts/lint-markdown.sh`
- `bash scripts/validate-qml.sh`
- Validação de sintaxe YAML dos arquivos GitHub
